# Git Commit 中文规范

## 核心原则
在生成 Git commit 信息时，必须使用中文编写提交信息。

## Commit 信息格式

### 基本结构
```
<类型>: <简短描述>

<详细说明>（可选）
```

### 提交类型（中文）
- **功能**: 新增功能
- **修复**: 修复 bug
- **文档**: 文档更新
- **样式**: 代码格式调整（不影响功能）
- **重构**: 代码重构（不改变功能）
- **性能**: 性能优化
- **测试**: 添加或修改测试
- **构建**: 构建系统或依赖更新
- **配置**: 配置文件修改
- **初始化**: 项目初始化

## 示例

### 单行提交
```
功能: 添加股票搜索功能
修复: 解决 K 线图数据加载问题
文档: 更新 README 安装说明
重构: 优化指标计算器代码结构
```

### 多行提交
```
功能: 实现技术指标计算功能

- 添加 MA、EMA、MACD 等指标计算
- 实现指标数据缓存机制
- 添加单元测试覆盖
```

## 注意事项
- 使用简洁明了的中文描述
- 第一行不超过 50 个字符
- 详细说明使用列表形式，便于阅读
- 避免使用英文缩写，除非是通用技术术语（如 API、SQL）

## 适用范围
- 所有 Git commit 操作
- 自动生成的提交信息
- 用户请求生成的提交信息
